# [PRD] ggotAIya 제품 요구사항 문서 (Product Requirement Document)

## 1. 프로젝트 개요 (Project Overview)

* 
**앱 이름:** ggotAIya (꽃아이 야) 


* 
**한줄 설명:** `ggotAIorder`가 수집·입력한 6대 채널의 주문 접수 현황을 실시간 모니터링하고, 누적 주문 조회, 알림 템플릿 설정 및 백그라운드 서비스의 가동을 직접 제어하는 1920*1080 해상도 최적화 포그라운드 대시보드 애플리케이션. 


* 
**개발 유형:** 데스크톱 포그라운드 애플리케이션 (웹 프론트엔드 아키텍처 기반 코어 패키징) 


* **개발 난이도:** **중상 (Medium-High)**
* *이유: 백그라운드 윈도우 서비스(`ggotAIorder`)를 포그라운드 UI에서 명령어 인터페이스로 강제 켜고 끄는 프로세스 제어 연동 기술, Supabase Realtime 웹소켓을 바인딩하여 새로고침 없이 화면의 작동 원(🟢/🔴)과 현재 작업 상태 문자열을 꺾어주는 실시간 데이터 동기화, 쇼핑몰/인터라넷 패스워드를 양방향 AES-256 알고리즘으로 안전하게 암호화·복호화하는 보안 처리 메커니즘이 포함되기 때문임.*



## 2. 사용자 시나리오 (User Scenario)

* 
**누가 (Who):** 매장 컴퓨터 화면에 항상 관리창을 띄워두고 현장 작업과 주문 현황 관리를 동시에 조율해야 하는 꽃가게 사장님 


* 
**언제 (When):** * 매장 출근 후 컴퓨터를 켜고 오늘 자 주문 수집 시스템을 가동하거나 중지하고 싶을 때 


* 작업 도중 스마트폰이나 전화로 들어온 주문이 전산에 어떤 단계(STT 분석, Gemini 분석, RPA 입력 등)로 처리되고 있는지 실시간으로 모니터링하고 싶을 때 


* 기간별, 채널별로 누적된 정형 주문 데이터와 금액 합계를 정밀하게 정산·조회하고 싶을 때 


* 쇼핑몰/인터라넷 계정 정보가 바뀌었거나 고객에게 발송될 자동 보고 알림톡(성공/실패) 문구를 수정하고 싶을 때 




* 
**왜 (Why):** * `ggotAIorder`는 화면이 없는 백그라운드 서비스이기 때문에 시스템이 지금 살아있는지, 입력 오류가 나진 않았는지 눈으로 확인할 길이 없음. 


* 따라서 사장님이 "한 화면에서 모든 채널의 처리 현황을 실시간 눈으로 확인하여 심리적 안정감을 얻고, 계정 설정이나 알림톡 제어 등을 마우스 클릭 몇 번으로 직관적으로 통제"할 수 있는 통합 컨트롤러 화면이 반드시 필요하기 때문임. 





## 3. 핵심 기능 목록 (Core Features)

* F1. 수동 회원가입 및 정보 적재: 사장님이 PC 화면에서 꽃집 정보, 아이디(`username`), 패스워드를 수동 입력하여 `member_info` 테이블에 신규 데이터를 안전하게 적재하는 기능 


* F2. 중복 아이디 및 핸드폰 인증 제어: 가입 시 아이디 중복 확인 및 실시간 핸드폰 인증 절차를 거치도록 통제하는 보안 기능 


* F3. ID/PW 기반 수동 로그인 및 세션 보존: 아이디와 비밀번호를 직접 입력하여 접속하며, '자동로그인' 체크 시 쿠키나 토큰 형태로 PC에 세션을 영구 보존하는 기능 


* F4. 핸드폰 인증 기반 계정 찾기: 아이디 및 핸드폰 실시간 인증 번호 확인을 통해 분실된 정보를 안전하게 화면에 표출해 주는 수동 찾기 기능 


* F5. 마이페이지 회원 정보 관리: 가입된 사장님 정보 중 아이디를 제외한 꽃집명, 주소, 대표자 등의 정보를 핸드폰 재인증 후 안전하게 수정(`UPDATE`)하는 기능 


* F6. 6대 채널 주문접수 상황판 표출: 가게전화, 핸드폰1, 핸드폰2, 가게음성, 쇼핑몰, 인터라넷의 실시간 가동 상태와 카운트를 바인딩하는 대시보드 뷰 기능 


* F7. Supabase Realtime 웹소켓 실시간 UI 갱신: 화면 새로고침 없이 `server_call_history`와 `order_details` 테이블의 변화를 감지하는 즉시 작동 원(🟢/🔴)과 현재 작업 상태 문자열을 동적으로 꺾어주는 실시간 바인딩 기능 


* F8. 윈도우 백그라운드 서비스 명령 제어: 상황판 상단 제어 버튼 클릭 시 OS 명령어를 인터페이스하여 백그라운드 서비스(`ggotAIorder`)를 강제로 시작(`net start`)하거나 중지(`net stop`)시키는 마스터 스위치 기능 


* F9. 당일 RPA 처리 누적 건수 실시간 반영: 금일 `order_details` 테이블에 적재되어 RPA 입력까지 최종 마감된 성공 건수를 실시간 스캔하여 스코어보드에 뿌려주는 기능 


* F10. 다중 채널 필터링 및 기간별 주문 검색: '전체' 또는 특정 채널을 타겟팅하고, 사장님이 지정한 일자 매개변수 범위 내의 주문서 데이터를 쿼리해 오는 기능 


* F11. 정형 데이터 그리드 표출 및 상태 가시화: 수집된 주문서 리스트를 그리드 형태로 정렬하고, 각 주문의 진행 상태(`대기`, `성공`, `실패`)를 한눈에 식별할 수 있도록 가시화하는 기능 


* F12. 실시간 매출 합계 및 데이터 페이징: 검색된 결과의 총 건수와 주문 금액 총합을 자동 연산하여 하단에 요약 노출하고, 대량 데이터 처리를 위한 페이징 네비게이션 기능 


* F13. 알림톡/문자 수신처 및 발송 토글 설정: 알림 발송 여부 스위치 및 보고를 수신할 번호를 지정·수정하는 기능 


* F14. 메시지 템플릿 실시간 수정 및 매핑: `{channel}`과 `{count}` 변수가 포함된 성공/실패 보고 문구를 사장님 입맛에 맞게 커스텀하여 `setting_info`에 반영하는 기능 


* F15. 쇼핑몰/인터라넷 계정 설정 및 스케줄러 간격 제어: 크롤링할 외부 사이트의 주소, ID 정보와 수집 주기(분 단위 간격 정수)를 설정하는 기능 


* F16. 패스워드 비밀 가시화 및 양방향 AES-256 암호화 보안: 패스워드 필드는 기본적 `'*'`로 숨기되 눈 모양 클릭 시 임시 복호화하여 가시화하며, [수정] 완료 시에는 무조건 AES-256 알고리즘으로 암호화하여 DB에 밀어 넣는 강력한 보안 기능 



## 4. 기술 스펙 (Technical Stack)

* **데스크톱 패키징 환경:** `Electron` (Chromium 엔진 기반 반응형 웹 구동 런타임)
* **화면 프레임워크:** `React.js` (컴포넌트 상태 기반 UI 렌더링)
* **반응형 스타일링 라이브러리:** `Tailwind CSS` & `CSS Grid / Flexbox` (반응형 접두사 및 자석형 레이아웃 구성)
* **중앙 백엔드 연동 SDK:** `Supabase JavaScript SDK`
* **실시간 UI 동기화 엔진:** `Supabase Realtime` (웹소켓 기반 0초 데이터 갱신)
* 
**양방향 보안 암호화:** `crypto-js` (클라이언트단 패스워드 **AES-256 알고리즘** 암호화 모듈) 


* **OS 프로세스 명령어 제어:** Node.js `child_process` (윈도우 서비스 `net start` / `net stop` 명령어 제어용)

## 5. 화면 구성 (UI Components)

### 5-1. 공통 네비게이션 헤더 (Common Header)

* 
**로그인 전 헤더:** [주문접수상황판], [주문접수조회], [환경설정] 비활성화 뷰, 우측 최상단 **[로그인]**, **[회원가입]** 단추 배치. 


* 
**로그인 후 헤더:** 우측 상단에 `member_info` 기반 **`[가게명(예: 서울꽃집)]`**, **`[마이페이지]`**, **`[로그아웃]`** 인터페이스 정렬. 



### 5-2. 사용자 인증 화면군 (로그인 전 접속 가능)

* 
**[화면 1] 홈 화면 (로그인 전 대기 상태):** 중앙 메인 영역에 홍보 메시지와 **'ggotAI 브랜드를 상징하는 이쁜 로봇 마스코트 로고'** 가 반응형으로 배치됨. 


* 
**[화면 2] 수동 로그인 화면:** `username`(아이디), `password`(비밀번호) 입력 필드, **[자동로그인] 체크박스**, 하단 **[아이디 찾기] / [비밀번호 찾기]** 팝업 링크. 


* 
**[화면 3] 아이디 찾기 화면:** `[꽃가게 이름]`, `[핸드폰번호]` 입력창 및 **[아이디 찾기]** 실행 버튼. 


* 
**[화면 4] 비밀번호 찾기 화면:** `[아이디]`, `[핸드폰번호]`, `[인증번호]` 입력 및 **[확인]** 단추, **[비밀번호 찾기]** 최종 버튼. 


* 
**[화면 5] 회원가입 화면:** `[아이디]` + **[중복확인]**, `[비밀번호]` 및 확인란, `[꽃집명]`, `[대표자]`, `[전화]`, `[핸드폰]` + **[인증]**, `[이메일]`, `[주소]` + **[주소찾기]** API 및 `[상세주소]` 입력창, 최하단 **[회원가입]** 마감 버튼. 



### 5-3. 대시보드 및 관리 화면군 (로그인 후 전용)

* 
**[화면 6] 홈(Home) 화면 - 실시간 주문접수 상황판:** 


* 
**마스터 가동 제어부:** 가동 상태 문구(`주문수집중` / `주문수집 중지`)와 가동을 제어하는 **`[주문수집 중지]` 또는 `[주문수집 시작]` 마스터 스위치**. 


* 
**6대 채널 반응형 그리드 상황판 테이블:** [가게전화, 핸드폰1, 핸드폰2, 가게음성, 쇼핑몰, 인터라넷] 6대 영역 배열. 


* 
**`작동` 행:** 유효성 상태에 따라 **'녹색원'** 또는 **'빨간원'** 점등. 


* 
**`현재작업` 행:** 파이프라인 진행 상태(`STT 분석중`, `주문정보 분석중`, `화원관리 입력중`, `대기`, `미사용`) 동적 갱신. 


* 
**`오늘작업` 행:** 금일 전산 자동 입력까지 완벽하게 처리 완료된 누적 성공 수량(`X건`) 출력. 






* 
**[화면 7] 마이페이지 화면:** `[아이디]` 필드는 수정 불가 마킹, 정보 입력 후 `[핸드폰 인증]` 통과 시 가동되는 **[수정]** 버튼 배치. 


* 
**[화면 8] 주문접수 조회 화면:** 


* 
**상단 필터 바:** 채널 분류 세그먼트(`전체`, `핸드폰`, `가게전화`, `쇼핑몰`, `인터라넷`, `가게음성`), 기간 필터, **[조회]** 버튼. 


* 
**메인 데이터 그리드:** `No`, `주문일자`, `주문시간`, `고객명`, `상품`, `가격`, `배달장소`, `채널`, `입력상태` (`대기`, `성공`, `실패` 뱃지 적용) 리스트 표출. 


* 
**하단 요약 바:** 검색 데이터 범위 내 총 주문 건수 및 총 금액 합계 요약 볼드 노출. 




* 
**[화면 9] 환경설정 화면:** 


* 
**메시지 전송 블록:** `[카카오톡]` / `[문자]` 토글 스위치, 수신 핸드폰 번호 입력 폼, `{channel}`, `{count}` 내장형 `[성공메세지]` / `[실패메세지]` 입력창. 


* 
**주문 전화 매핑 블록:** `핸드폰1`, `핸드폰2`, `일반전화1`, `일반전화2` 수동 기재 창. 


* 
**외부 연동 블록:** `[쇼핑몰 주소/ID/PW]` 및 `[인터라넷 주소/ID/PW]` 입력 컴포넌트 (눈 모양 비밀 가시화 아이콘 필수 탑재). 


* 
**간격 제어 블록:** `[쇼핑몰 주문 확인 시간 간격(분)]`, `[인터라넷 시간 간격(분)]` 기입 필드. 


* 
**최하단 제어 스위치:** 환경 정보를 영구 마감 처리하는 **[수정]** 버튼 배치. 





## 6. 상세 기능 명세 (Functional Specifications)

### 6-1. [인증 및 가입 모듈] 엔터 키 포커스 및 수동 가입 제어

* 
**키보드 내비게이션 룰:** 마우스를 잡지 않고 신속하게 입력할 수 있도록, 모든 입력 폼에서 **키보드 엔터(Enter) 키를 누르면 다음 입력 필드로 포커스가 자동 이동**하는 이벤트를 구현함. 


* 
**아이디 중복 확인 및 가입 통제:** 가입 시 `[중복확인]` 클릭 즉시 `member_info` 테이블을 조회하여 중복 존재 시 경고 메시지를 뿜고 가입을 차단함. 


* 
**세션 유지 자동 로그인:** 로그인 성공 후 `[자동로그인]` 체크 시 쿠키/토큰 세션을 로컬에 영구 보존하여 다음 구동 시 로그인 창을 패스하고 홈 화면으로 다이렉트 진입시킴. 



### 6-2. [대시보드 모듈] 실시간 수신 웹소켓 및 상태 결합 UI 갱신

* 
**Supabase Realtime 웹소켓 연동:** `Supabase Realtime` 채널을 활성화하여 수동 새로고침 없이 `server_call_history`와 `order_details` 테이블의 변화(`INSERT`, `UPDATE`)를 실시간 포착함. 


* **상태 결합 기반 '현재작업' 문자열 동적 변환 가이드:**
* 상황판의 **`[현재작업]`** 열 텍스트는 `server_call_history.is_order` 필드와 `order_details.rpa_status` 상태 값을 결합하여 다음과 같이 실시간 정의함. 




1. 음성 데이터 유입 후 로컬 STT 엔진 가동 중 ➔ **`"STT 분석중"`** 


2. STT 완료 후 Gemini LLM이 가공 분석 처리 중 ➔ **`"주문정보 분석중"`** 


3. Gemini 통과 후 `order_details`에 정형 적재되어 RPA 대기 중인 상태 (`rpa_status='ready'`) ➔ **`"대기"`** 


4. RPA 로봇이 윈도우 전산 프로그램을 붙잡고 타이핑 입력 중인 상태 ➔ **`"화원관리 입력중"`** 


5. 전산 프로그램 입력이 끝나 `rpa_status='success'`로 마감 처리된 순간 ➔ **`"화원관리 입력완료"`** 




* **하드웨어 및 네트워크 예외 상태 가시화:**
* 
`setting_info` 내 필수 주소값 누락, DB 에러, 전화기 전원 꺼짐, 외부 세션 만료 등의 장애 수집 불가 사유가 감지되면 ➔ 상황판 `[작동]` 칸에는 즉시 '빨간원(Red)'을 키고, `[현재작업]` 칸에는 빨간색 텍스트로 `'미사용'`, `'전화기 꺼져있음'`, `'접속에러'` 등 명확한 원인을 노출함. 





### 6-3. [대시보드 모듈] 윈도우 서비스(net start/stop) 마스터 스위치 제어

* **주문수집 중지 액션:** 사장님이 **`[주문수집 중지]`** 버튼을 누르면, Electron 내부 Node 프로세스 핸들러가 관리자 권한으로 **`net stop ggotAIorder`** 시스템 명령어를 강제 실행함. 동시에 상황판 모든 채널 불빛을 빨간원으로 꺾고 상태 문구를 **`"중지"`** 로 일괄 업데이트함. 


* 
**주문수집 시작 액션:** 사장님이 **`[주문수집 시작]`** 버튼을 누르면, 반대로 **`net start ggotAIorder`** 명령어를 실행하여 백그라운드 수집 서비스를 상주 대기 상태로 복구함. 



### 6-4. [조회 모듈] 그리드 필터링 및 실시간 정산 매출 연산

* 
**채널별 탭 필터링:** 필터바에서 세그먼트 클릭 시 데이터베이스 조인 쿼리 매개변수를 자동 변경하여 리스트 그리드를 새로고침함. 


* 
**정산 금액 실시간 롤업 연산:** 사장님이 지정한 날짜 매개변수 범위 내에서 표출된 모든 주문 행들의 수량과 가격을 곱하여, 전체 누적 주문 총 건수와 **최종 매출 금액 합계(예: `490,000원`)를 자동 연산**하여 하단 요약 바에 볼드로 실시간 출력함. 



### 6-5. [환경설정 모듈] 입력값 디폴트 연동 및 양방향 AES-256 패스워드 암호화

* 
**회원 정보 기본값(Default) 로딩:** 초기 로딩 시 `[카카오톡/문자]` 수신 번호와 `[핸드폰1]`, `[일반전화1]` 필드에 `member_info` 테이블의 `mobile_number`와 `landline_number` 값을 기본 디폴트 값으로 매핑 로드함. 


* 
**패스워드 은닉 및 가시화:** 쇼핑몰 및 인터라넷 패스워드는 상시 마스킹 처리하되, 우측 끝단의 **'눈 모양 아이콘'을 마우스로 누르고 있는 동안에만 평문으로 임시 가시화**해 줌. 


* 
**★ [핵심 보안] 양방향 AES-256 암호화 통신:** 환경설정창에서 **`[수정]`** 버튼을 클릭하는 순간, 프론트엔드는 패스워드 평문을 그대로 네트워크에 흘리지 않고 `crypto-js` 라이브러리를 가동하여 **AES-256 알고리즘으로 암호화 처리를 완료한 후 Supabase DB(`setting_info`)에 최종 안전 저장**함. 



## 7. 디자인 가이드 (Design Guide)

* 
**반응형 룰 (Breakpoints):** `Tailwind CSS` 레이아웃 프리셋을 전면 적용하여 대화면 PC 해상도(1920이상)에서는 가로 6열 테이블 전광판 뷰로 보이고, 창 모드 및 소화면에서는 컴포넌트가 흐트러지지 않고 자석처럼 1열 카드(Card) 형태로 자동 정렬·세로 쪼개짐 처리됨. 


* 
**기본 디자인 심볼 (홈 화면):** 로그인 전 홈 화면 중앙에 **'ggotAI 브랜드를 상징하는 이쁜 로봇 마스코트'** 이미지를 반응형 크기로 중앙 배치함. 


* **상태별 색상 시스템:**
* 🟢 **녹색원 (`#10B981`):** 수집 가동이 활성화된 정상 수집 상태. 


* 🔴 **빨간원 (`#EF4444`):** 설정 누락, DB 에러, 서비스 정지 등으로 인한 수집 불가능 상태. 


* 
**진행/성공 상태:** 녹색/파란색 뱃지 테마 적용 (`대기`, `성공`, `화원관리 입력완료`). 


* 
**장애/실패 상태:** 선명한 빨간색 볼드 텍스트 및 뱃지 적용 (`미사용`, `전화기 꺼져있음`, `실패`, `중지`). 





## 8. 제약 사항 (Constraints)

* 8-1. UI 환경 및 반응형 제어 제약: 대화면 전광판 형태의 1920*1080 해상도를 최적 규격으로 하되 창 크기 변형 시 자석처럼 카드로 쪼개지는 반응형 그리드 구조를 강제하며, 모든 입력창에서 엔터(Enter) 키 클릭 시 다음 포커스로 자동 이동되는 인터랙션 코드를 무조건 보장해야 함. 


* 8-2. 데이터 동기화 및 실시간 구독 제약: 화면 내 수동 새로고침(`F5`) 및 타이머 호출 방식을 차단하고, 반드시 `Supabase Realtime` 웹소켓 바인딩을 통해 데이터 상태 변화를 0초 만에 인터셉트하여 화면을 갱신해야 함. 


* 8-3. 강력한 보안 및 암호화 제약: 외부 연동 패스워드 저장 시 무조건 클라이언트단에서 `crypto-js` 기반 **AES-256 알고리즘 암호화**를 거친 후 데이터베이스에 적재해야 하며, 화면 표출 시 눈 모양 단추 클릭 타이밍에만 복호화 임시 가시화 제어를 수행해야 함. 


* 8-4. OS 커널 제어 권한 제약: UI 앱 내부에서 `net start` / `net stop` 프로세스 핸들러 명령을 커널에 직접 날려 윈도우 백그라운드 서비스의 생명주기를 통제해야 하므로, 본 프로그램은 반드시 '윈도우 관리자 권한(Administrator Privileges)'으로 가동되어야 함. 


* 8-5. 로그아웃 시 서비스 독립성 보장: 사장님이 UI 앱에서 `[로그아웃]`을 누르거나 대시보드 창을 끄더라도, 백단에서 돌고 있는 수집 엔진(`ggotAIorder`) 프로세스는 연쇄 종료되지 않고 독립 구동 상주해야 함.
